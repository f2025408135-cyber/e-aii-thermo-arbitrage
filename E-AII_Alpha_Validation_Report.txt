The friction-hardened simulation processes the full 104,832-tick dataset (5-minute intervals, 12-month period)

(cid:127) STEP 1 - Stochastic Execution Latency: Each signal receives a random execution delay drawn from a

slippage is deducted from the raw spread, and if the net spread goes negative, the trade is classified as a
loss.
(cid:127) STEP 3 - Thermal Inertia Overhang: The thermal module runs continuously (even during non-trading ticks),

11.1 Hyper-Local Meteorological Nowcasting
The nowcasting module uses gradient-boosted trees (GBT) and long short-term memory (LSTM) networks
trained on 5-minute lag features from the historical telemetry database. The target variable is the M_drift value
30 minutes (6 ticks) ahead. The feature set includes: current and lagged wind speed (6 lags), current and
lagged AHF (6 lags), M_drift momentum (1-lag and 6-lag differences), diurnal hour-of-day encoding, and rolling
statistics (1-hour and 6-hour windows). The target performance metric is MAPE < 5.5% on the 30-minute ahead
prediction, which would allow the system to detect and pre-position for M_drift spikes well before they occur.
11.2 PRE_FLIGHT_WARMUP State Machine
When the predicted M_drift (30 minutes ahead) exceeds 1.50, the system transitions from the IDLE state to the
PRE_FLIGHT_WARMUP state. In this state, the system: (1) pre-allocates GPU compute resources on both
RENDER and Vast.ai venues, (2) submits limit orders at the current mid-price with 30-minute time-in-force, (3)
establishes API connections and warms up the execution pipeline, and (4) monitors the real-time M_drift feed
for confirmation. When the actual M_drift crosses the 1.10 thermal wall threshold, the pre-positioned orders are
already in the market, eliminating the 30-second execution latency entirely. The system then transitions to the
ACTIVE state for position management and exit.
11.3 Projected Impact on Win Rate
By eliminating the 30-second execution latency, the nowcasting MPC is projected to increase the win rate from
56.47% to approximately 85%+. The reasoning is as follows: of the 6,212 losing trades in the friction-hardened
simulation, approximately 85% (5,280) were trades where the raw spread was positive at detection time but

residual prediction errors (MAPE 5.5%) and the possibility that some pre-positioned orders may not fill due to
market microstructure changes during the 30-minute prediction window.
